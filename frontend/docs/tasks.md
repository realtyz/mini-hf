# 任务队列

本文档介绍如何通过 Web 界面创建和管理下载任务。

## 任务概述

下载任务用于将 HuggingFace 仓库缓存到本地服务器。任务创建后，系统会自动从 HuggingFace 下载文件并存储到本地存储中。

## 任务状态

| 状态 | 说明 |
|------|------|
| 等待审批 | 任务已创建，等待管理员审批（大容量任务） |
| 排队中 | 任务已进入队列，等待执行 |
| 进行中 | 任务正在执行，正在下载文件 |
| 已完成 | 任务执行成功，仓库已缓存 |
| 已失败 | 任务执行失败，请查看错误信息 |
| 已取消 | 任务被用户或管理员取消 |

## 创建下载任务

### 1. 进入任务页面

登录系统后，点击左侧导航栏的「任务列表」进入任务管理页面。

### 2. 点击「新建任务」

在任务列表页面点击右上角的「新建任务」按钮。

### 3. 填写任务信息

| 字段 | 说明 |
|------|------|
| 仓库来源 | 目前支持 HuggingFace |
| 类型 | 选择「模型」或「数据集」 |
| 仓库ID | HuggingFace 仓库标识符，如 `bert-base-uncased` |
| 版本/分支 | 要下载的版本，默认 `main` |
| 访问令牌 | 私有仓库需填写 HuggingFace 访问令牌 |
| 全量下载 | 勾选后下载所有文件 |

### 4. 文件过滤（可选）

如只需下载部分文件，可取消勾选「全量下载」，然后配置过滤规则：

**允许模式 (allow_patterns)**：仅下载匹配的文件

```
*.bin          # 下载所有 .bin 文件
models/**      # 下载 models 目录下所有文件
*.safetensors  # 下载所有 safetensors 文件
```

**忽略模式 (ignore_patterns)**：排除匹配的文件

```
*.md           # 忽略所有 Markdown 文件
*.gif          # 忽略所有 GIF 图片
```

### 5. 预览并确认

点击「下一步」后，系统会获取仓库文件列表并显示预览信息：
- 待下载文件数量
- 所需存储空间
- 文件列表

确认无误后点击「确认创建」。

## 审批流程

当任务所需存储空间超过一定阈值时，需要管理员审批：

1. 任务创建后进入「等待审批」状态
2. 管理员收到通知，审核任务
3. 管理员批准后，任务进入「排队中」状态
4. 如被拒绝，任务变为「已取消」状态

## 任务管理

### 查看任务详情

点击任务行可查看详细信息：
- 下载进度
- 文件列表及状态
- 错误信息（如有）

### 取消任务

对于排队中或进行中的任务，可点击「取消」按钮终止任务。

### 刷新列表

点击「刷新」按钮可获取最新的任务状态。

## Glob 模式语法

文件过滤使用 glob 模式匹配：

| 模式 | 说明 | 示例 |
|------|------|------|
| `*` | 匹配任意字符（不含 `/`） | `*.bin` 匹配所有 .bin 文件 |
| `**` | 匹配任意字符（含 `/`） | `**/*.json` 匹配所有目录下的 .json 文件 |
| `?` | 匹配单个字符 | `model-?.bin` 匹配 model-1.bin |
| `[abc]` | 匹配指定字符之一 | `model-[12].bin` |

**注意**：模式不应以 `/` 开头或结尾，且不能包含 `..` 目录遍历符。
